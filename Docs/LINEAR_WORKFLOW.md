# Linear Workflow

## Required Fields

- Issue key (for example `IGR-101`) must be referenced in:
  - branch name,
  - commit messages,
  - world changelog entry (if world changed).

## Status Flow

1. `Backlog`
2. `Ready`
3. `In Progress`
4. `Review`
5. `QA`
6. `Done`

## Recommended Labels

- `server`
- `client`
- `shared`
- `world`
- `ui`
- `bug`
- `tech-debt`

## Definition of Done

1. Linked code changes merged.
2. Studio behavior verified.
3. `WORLD_CHANGELOG.md` updated for world-only changes.
4. Issue moved to `Done`.
